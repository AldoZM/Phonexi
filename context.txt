# Phonexi — Project Context

## What it is
Background Python daemon for Windows. Listens for Right Shift + P hotkey. On trigger:
1. Captures screenshot of the monitor where the cursor is
2. Sends screenshot to Groq vision LLM
3. Streams response into a dark terminal-style Tkinter popup on secondary monitor

Primary use case: discreet coding assistant during interviews or problem-solving sessions.

---

## Tech Stack
- Python 3.14 on Windows 10
- pynput — global hotkey detection (Right Shift + P, case-insensitive)
- mss + ctypes.windll.user32.GetCursorPos — per-monitor screenshot capture
- groq (Python SDK) — LLM inference via Groq API (model: meta-llama/llama-4-scout-17b-16e-instruct)
- Tkinter Toplevel — dark popup UI (overrideredirect=True, no taskbar entry)
- pygments — syntax highlighting inside code blocks
- python-dotenv — load GROQ_API_KEY from .env

## Key Design Decisions

### Threading model
- pynput listener runs in daemon thread (blocks if on main thread)
- Tkinter mainloop runs on main thread
- Widget creation must happen on main thread → use root.after(0, callback)
- LLM streaming runs in worker daemon thread

### Hotkey detection
- pynput reports uppercase 'P' when Shift is held → check k.char.lower() == 'p'
- Modifier tracked via set() of pressed keys + threading.Lock

### Screenshot
- mss monitors[0] = combined virtual screen (wrong)
- monitors[1] = primary, monitors[2+] = secondary
- Use ctypes.windll.user32.GetCursorPos to find which monitor the cursor is on
- Capture only that monitor → saves as PNG in /screenshots/

### Popup window
- overrideredirect(True) → no title bar, no taskbar entry (discreet)
- Always appears on secondary monitor (monitors[2] if exists, else monitors[1])
- Closes with Escape key
- -topmost True to stay above other windows

### LLM response rendering
- Buffer all streamed tokens first ("Analyzing screenshot..." shown meanwhile)
- After full response received → parse Markdown → render with tags
- Code blocks (```lang ... ```) → pygments syntax highlighting (Dracula color scheme)
- Headings, bold, italic, inline code → Tkinter text tags

### API key security
- GROQ_API_KEY stored in .env only (never committed)
- .gitignore excludes .env file explicitly

## Why Groq instead of Google Gemini
Originally planned to use Google Gemini API (google-genai SDK). Failed because:
- User's Google account (associated with an organization) generates AQ.Ab8... format keys
  instead of standard AIza... API keys
- These are OAuth2/service-account-bound tokens, not standard API keys
- quota=0 for free tier on gemini-2.0-flash for the gen-lang-client project
- Google Cloud Console forces service account binding for all Gemini API credentials
Groq was chosen as alternative: free tier, standard API key, fast inference, vision support.

## Setting Up on a New Machine

### Prerequisites
- Windows 10/11 (required — uses ctypes.windll, pynput, Tkinter)
- Python 3.10+ (tested on 3.14)
- Git

### Step-by-step

```bash
# 1. Clone the repo
git clone https://github.com/AldoZM/Phonexi.git
cd Phonexi

# 2. (Recommended) Create a virtual environment
python -m venv .venv
.venv\Scripts\activate        # Windows CMD
# or: .venv/Scripts/Activate.ps1   # PowerShell

# 3. Install dependencies
pip install -r requirements.txt

# 4. Create the .env file with your Groq API key
#    .env is NOT in git — you must create it manually on every machine
echo GROQ_API_KEY=gsk_your_key_here > .env

# 5. Run
python main.py
```

### Getting a Groq API key
1. Go to https://console.groq.com
2. Create a free account (no credit card required)
3. API Keys → Create API key
4. Copy the key (starts with gsk_...)
5. Paste it into .env as shown above

### What NOT to do
- Do NOT commit .env — the API key must stay local
- Do NOT use Google Gemini — the owner's Google account is organization-managed
  and only generates service-account-bound OAuth tokens (AQ.Ab8... format),
  not standard API keys. Groq is the chosen provider.
- Do NOT run on Linux/macOS — ctypes.windll and pynput global hotkeys
  require Windows

### Troubleshooting
| Symptom | Cause | Fix |
|---------|-------|-----|
| `GROQ_API_KEY not set` error | .env missing or empty | Create .env with key |
| Popup appears on primary monitor | Only 1 monitor detected | Connect second monitor |
| Hotkey not detected | Another app captured Right Shift+P | Close conflicting app |
| `ModuleNotFoundError` | venv not activated or deps not installed | Run `pip install -r requirements.txt` |
| `GroqAPIError` 429 | Daily rate limit hit (14,400 req/day free) | Wait until next UTC day |
